> **Abstract:** This paper introduces Agent-to-User Interfaces (A2UI), a deterministic interface model that transforms technical content into machine-readable, executable knowledge systems, demonstrated through the PADI Sovereign Bureau.

## 1\. Philosophy: Epistemic Visibility as a Function of Schema

In the traditional record of technological progress, the "erasure" of marginalized pioneers is often framed as social or historical bias. From an Information Science perspective, it is more precisely understood as a **structural data failure**.

In agent-mediated environments, **epistemic visibility is a function of schema compliance**. If a contribution is not encoded into a machine-readable format, it becomes invisible to the autonomous systems that govern modern knowledge retrieval.

## 2\. The Problem: Structural Invisibility & Dark Data

Traditional knowledge systems rely on **popularity heuristics**, which distort attribution and suppress structurally underrepresented contributors.

Examples include pioneers such as:

*   Gladys West (GPS systems)
    
*   Alice Ball (medical chemistry)
    

### Key Failures:

*   **Schema Absence:** Narrative-only records cannot be ingested by knowledge graphs.
    
*   **Citation Fragility:** Narrative is lossy; structured data is persistent.
    
*   **Visibility Bias:** Influence becomes a function of searchability, not contribution.
    

Without structured encoding, technical equity remains a **non-computable concept**.

## 3\. The Framework: Agent-to-User Interfaces (A2UI)

A2UI represents a paradigm shift from passive dashboards to **Active Reasoning Interfaces**.

### A2UI as a Deterministic Interface Model

A2UI redefines the interface as a **Cognitive Boundary Layer where machine reasoning becomes externally inspectable**:

*   **Input:** Machine-readable artifacts (JSON-LD manifests)
    
*   **Processing:** Deterministic or probabilistic agent workflows
    
*   **Output:** Human-interpretable reasoning states
    

### The Three-Layer Architecture

```plaintext
Discovery Layer → Protocol Layer → Execution Layer  
     (UI)              (Schema)            (Compute)
```

1.  **Discovery Layer (Interface)** Human-facing platforms (Dev.to, Hashnode) embedding structured manifests.
    
2.  **Protocol Layer (Authority)** GitHub-based contracts (`AGENTS.md`, `ACTION.md`) defining identity and interaction rules.
    
3.  **Execution Layer (Compute)** Deterministic systems (Node.js, Render, Zeabur) performing:
    

*   Graph traversal (BFS)
    
*   Influence computation
    
*   Structural mapping
    

### Case Study: The PADI Sovereign Bureau

The **Living Library of Access** operationalizes A2UI as a structured knowledge system.

It currently indexes 23 pioneers across:

*   Computing
    
*   Justice
    
*   Medicine
    

### Structural Computation Model

Influence is computed—not narrated—using:

*   **Breadth-First Search (BFS):** Path-aware traversal
    
*   **Weighted Edge Propagation:** Temporal and technical influence modeling
    
*   **Category-Constrained Clustering:** Deterministic grouping
    
*   **Resistance to Popularity-Driven Ranking Bias**

## 5\. Protocol: Model Context Protocol (MCP)

The **Model Context Protocol (MCP)** is a structured interface standard that enables external agents to:

*   Interpret structured artifacts
    
*   Query knowledge systems
    
*   Execute workflows via semantic contracts
    

This enables **Machine-Readable Provenance** across distributed systems.
